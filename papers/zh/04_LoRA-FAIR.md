# LoRA-FAIR：具有聚合与初始化精化的联邦 LoRA 微调

**论文：** [arXiv 2411.14961](https://arxiv.org/abs/2411.14961)  
**发表于：** ICCV 2025  
**年份：** 2024（预印本）/ 2025（正式发表）  
**代码：** [GitHub](https://github.com/jmbian/LoRA-FAIR)  
**关键词：** 联邦 LoRA、聚合偏差、初始化漂移、PEFT

---

## 问题背景

朴素的联邦平均 LoRA 矩阵存在两个独立的失效模式：
1. **服务器端聚合偏差：** 平均来自不同客户端的 LoRA 矩阵 A 和 B 并不等于理想的全局更新，因为 LoRA 的更新是乘积形式（ΔW = BA），而不是线性叠加。对乘积取平均 ≠ 对平均值取乘积。
2. **客户端初始化滞后：** 每轮开始时客户端接收新的全局 LoRA，聚合后的初始化与本地最优点之间存在分布偏移，导致收敛缓慢。

## 核心思路

LoRA-FAIR 同时解决上述两个问题：
- 一种**基于残差的聚合**机制，纠正 LoRA 非线性乘积引入的聚合偏差。
- 一个**基于优化的初始化精化**步骤，在分发给客户端之前调整全局 LoRA，缩小初始化差距。

## 方法

- **聚合精化：** 对客户端 LoRA 矩阵取平均后，应用纠正步骤，最小化聚合乘积与目标理想更新之间的差异。
- **初始化精化：** 下一轮分发全局 LoRA 给客户端前，服务器端执行轻量级优化，使初始化与客户端预期起始点对齐。
- 两种精化操作均在 LoRA 参数空间内进行，不涉及完整模型权重，开销可控。

## 实验结果

- 在领域自适应基准上优于朴素 FedAvg + LoRA 及其他联邦 PEFT 基线。
- 客户端数据异构性越高，提升幅度越显著。
- 在单域准确率和总体平均准确率上均表现更优。

## 优点

- 诊断并修复联邦 LoRA 中两个独立的误差来源，而非只关注其中一个。
- 服务器端操作无需额外的客户端通信。
- 与现有 LoRA 变体兼容。

## 局限性

- 服务器端精化步骤增加了服务器的计算开销。
- 纠正依赖近似方法，在极高异构性下可能无法完全消除聚合偏差。

## 与个性化联邦学习的关联

尽管主要聚焦于提升全局模型质量，LoRA-FAIR 的聚合与初始化纠正同样使个性化微调受益——客户端在每轮开始时从一个更好对齐的全局状态出发，减少了用于纠正初始化误差的本地计算浪费。
