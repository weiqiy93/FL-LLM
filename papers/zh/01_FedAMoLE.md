# FedAMoLE：基于数据驱动异构模型架构的个性化联邦 LLM 微调

**论文：** [arXiv 2411.19128](https://arxiv.org/abs/2411.19128)  
**发表于：** ACM Web Conference 2026  
**年份：** 2024（预印本）/ 2026（正式发表）  
**关键词：** 个性化联邦学习、LoRA、混合专家（MoE）、模型异构性

---

## 问题背景

标准的个性化联邦 LLM 微调假设所有客户端共享相同的模型架构。这种统一结构无法适应客户端之间高度异构的数据分布，导致本地适应效果不佳。

## 核心思路

FedAMoLE 引入了**数据驱动的异构 LoRA 专家混合（AMoLE）模块**，每个客户端根据自身数据分布获得数量不同的 LoRA 域专家。框架还包含一种**反向选择专家分配（RSEA）策略**：不是客户端主动选择专家，而是域专家挑选与其知识领域最匹配的客户端，从而在训练过程中实现自动化架构调整。

## 方法

- 每个客户端在相同层上挂载异构的 LoRA 专家混合层，而非单一 LoRA 适配器。
- RSEA 策略让域专家识别最适合的客户端，减少知识聚合中的不匹配问题。
- 仅传输 LoRA 参数（而非完整模型权重），每轮通信开销约 25.66 MB（约为 FedMoE GB 级别的 1%）。
- 服务器通过匹配专家域（而非客户端身份）来聚合来自异构架构的 LoRA 专家。

## 实验结果

- 在 7 个联邦微调场景中进行评估。
- 相比现有个性化 FL 方法，客户端性能平均提升 **5.97%**。
- 与全量 MoE 基线相比，内存占用降低近 **50%**。
- 在模型异构的情况下，通信开销保持在实用范围内。

## 优点

- 消除了所有客户端必须共享相同模型架构的假设。
- 数据驱动的架构调整全自动完成，无需人工配置。
- 通过 LoRA 压缩实现高效通信。

## 局限性

- RSEA 策略增加了服务器端专家管理的复杂度。
- 异构聚合的性能依赖于预先定义足够多样化的域专家。

## 与个性化联邦学习的关联

本文是**架构层面个性化**的典型代表：每个客户端获得根据其本地数据量身定制的模型结构，超越了传统权重级别的个性化方式。
