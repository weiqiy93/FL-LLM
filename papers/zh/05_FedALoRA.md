# FedALoRA：面向 LLM 个性化联邦学习的自适应本地 LoRA 聚合

**论文：** [IEEE Xplore](https://ieeexplore.ieee.org/document/11048575) | [Springer](https://link.springer.com/chapter/10.1007/978-981-96-8731-2_9)  
**发表于：** WASA 2025（无线人工智能计算系统与应用会议）  
**年份：** 2025  
**关键词：** 个性化联邦学习、LoRA、非独立同分布、自适应聚合

---

## 问题背景

联邦 LLM 微调（FedLLM）面临**非独立同分布（non-IID）问题**的严峻挑战：客户端持有异构的本地数据分布（跨领域或跨任务），使得单一的全局聚合模型对任何个体客户端来说都是次优的。标准的 FedAvg LoRA 聚合将所有客户端适配器均匀平均，忽视了本地分布差异。

## 核心思路

FedALoRA 将**个性化模型聚合**与**基于 LoRA 的微调**相结合。服务器不再对所有客户端均匀聚合，而是通过按客户端相似度加权贡献，为每个客户端自适应地生成个性化全局模型。得到的个性化全局模型作为本地 LoRA 训练的初始化，使每个客户端在保留自身领域专业知识的同时，从相似同伴处获取通用知识。

## 方法

- **相似度加权：** 每轮开始前，服务器评估客户端两两相似度（如梯度余弦相似度或数据分布代理指标）。
- **自适应聚合：** 每个客户端接收一个定制化的全局 LoRA 适配器，对相似客户端的贡献权重更高，对差异较大的客户端权重较低。
- **本地 LoRA 训练：** 客户端以个性化全局 LoRA 为初始化对 LLM 进行微调，然后上传更新后的适配器。
- 全程在 LoRA 参数空间内操作，训练成本低。

## 实验结果

- 在跨领域非独立同分布设置下的大量实验表明，FedALoRA 优于标准 FedAvg + LoRA 及其他个性化 FL 基线。
- 在本地客户端任务上准确率更高，同时保留了通用能力。
- 由于 LoRA 参数量小，适合边缘端部署。

## 优点

- 个性化在聚合层面实现——无需修改模型结构。
- 易于在现有联邦 LoRA 流程上叠加实现。
- 自适应聚合能自然处理不同程度的客户端异构性。

## 局限性

- 估计客户端相似度需要共享部分梯度信息或代理指标，在某些场景下存在隐私隐患。
- 若客户端高度异构、不存在自然聚类，收益会下降。

## 与个性化联邦学习的关联

FedALoRA 实现了**聚合层面的个性化**：每个客户端的本地训练起点已根据其同伴邻域定制化，与非独立同分布数据下的个性化联邦微调需求高度契合。
